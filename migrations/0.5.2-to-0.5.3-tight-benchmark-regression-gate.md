# 0.5.2 to 0.5.3 - Tight Benchmark Regression Gate

## Summary

PAM 0.5.3 tightens the deterministic benchmark CI gate.

The benchmark check now compares current metrics against committed baselines and
allows at most a 1% regression by default. This replaces broad absolute
minimums that could miss meaningful performance or coverage losses.

## Action Required

No manual migration is required.

If your project intentionally changes benchmark behavior, update the committed
baselines in `tools/check-benchmark-thresholds.mjs` together with the benchmark
documentation.
