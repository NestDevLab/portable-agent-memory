# 0.5.3 to 0.6.0 - Agent Memory Fabric records

## Summary

PAM 0.6.0 adds the optional `amf-memory/v1` Markdown record contract, safe
graph projections, and two MCP tools for validation and proposal creation.

## Compatibility

No conversion of existing logs or graph-v1 data is required. Workspaces may
continue using classic PAM only. `memoryFormat` remains `graph-v1`.

## Adoption

1. Confirm `memory/pam.version.json` advertises `amfMemoryV1`.
2. Read `docs/amf-memory-v1.md`.
3. Create complete records through `memory_propose_record`.
4. Review the proposal artifact under `memory/maintenance/proposals/`.
5. Apply it with `memory_apply_proposal`.
6. Treat the returned graph node as a derived projection.

Person, relationship, confidential, and restricted records must be sealed
before proposal. PAM validates but does not manage encryption keys.

As of PAM 0.6.1 every `amf-memory/v1` record also requires exact structured
`confidence: {score,basis,assessedAt}` metadata. Before activating the 0.6.1
validator, records drafted against the initial 0.6.0 shape require the reviewed
offline bootstrap described in `0.6.0-to-0.6.1-amf-confidence.md`; afterward,
missing confidence fails closed. Confidence changes require a newer assessment
timestamp. Sealed revisions must refresh AAD and IV.
