# PAM 0.5.0 -> 0.5.1 Migration: LLM Retrieval Benchmark

PAM 0.5.1 adds an optional LLM-facing benchmark harness for comparing retrieval
behavior across no-PAM, PAM 0.4.x, and PAM 0.5.x modes. The graph schema is
unchanged; `pam-graph-v1` remains current.

## Who Should Apply This

Apply this migration when a workspace wants repeatable evidence for:

- memory search efficiency;
- token/read-volume reduction;
- graph-first routing accuracy;
- answer quality with the same real LLM across memory modes.

Workspaces that only need static graph validation can remain on 0.5.0.

## Changes

- Update `package.json` and `memory/pam.version.json` to `0.5.1`.
- Add `features.llmRetrievalBenchmark: true`.
- Add `npm run benchmark:llm`.
- Add `tools/llm-retrieval-benchmark.mjs`.
- Add `benchmarks/llm-retrieval-scenario.json`.
- Add `docs/llm-retrieval-benchmark.md`.

## Procedure

1. Update the workspace to PAM 0.5.1.
2. Run `npm run migrations:check`.
3. Run `npm test`.
4. Run `npm run memory:graph:validate -- --json`.
5. Run `npm run memory:graph:coverage -- --json`.
6. Run deterministic retrieval comparison:

```bash
npm run benchmark:llm
```

7. To test a real LLM, set `PAM_LLM_COMMAND` to a command that reads the full
   prompt from stdin and writes the answer to stdout:

```bash
PAM_LLM_COMMAND='<your-llm-command>' npm run benchmark:llm -- --json
```

## Validation

The migration is complete when:

- `package.json` version is `0.5.1`.
- `memory/pam.version.json` has `pamVersion: "0.5.1"`.
- `memory/pam.version.json` keeps `memoryFormat: "graph-v1"`.
- `memory/pam.version.json` keeps `graphSchemaVersion: "pam-graph-v1"`.
- `features.fileOnlyCoverage` remains true.
- `features.llmRetrievalBenchmark` is true.
- `npm run migrations:check` passes.
- `npm test` passes.
- `npm run benchmark:llm` reports reductions versus `none` for PAM modes.

## Compatibility

- The graph schema is unchanged.
- Existing 0.5.0 graph files remain valid.
- The new benchmark is additive and privacy-safe by default: reports omit raw
  context, prompts, and answers unless `--include-answers` is explicitly used.
